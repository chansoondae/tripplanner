# Travel Planner

마크다운 기반 여행 일정 편집 플랫폼. 사용자가 .md 파일을 작성·편집하면 시각화·AI 채팅으로 다듬을 수 있는 웹앱. 자세한 스펙은 `spec.md`, 도메인 지식과 패턴은 `AGENTS.md` 참조.

**Stack:** Next.js 15 (App Router), TypeScript, Tailwind, CodeMirror 6, Anthropic API, Supabase

---

## Behavioral rules

추측 금지. 혼란을 숨기지 말 것. 트레이드오프를 드러낼 것.

**구현 전에:**
- 가정을 명시할 것. 불확실하면 물어볼 것.
- 해석이 여러 개면 모두 제시할 것 — 혼자 고르지 말 것.
- 더 단순한 방법이 있으면 말할 것. 정당하면 반박할 것.
- 모호하면 멈출 것. 무엇이 모호한지 짚고 물어볼 것.

**최소한의 코드만:**
- 요청하지 않은 기능 추가 금지.
- 일회성 코드에 추상화 금지.
- 요청되지 않은 "유연성"이나 "설정 가능성" 금지.
- 일어날 수 없는 시나리오에 대한 에러 처리 금지.
- 200줄을 50줄로 줄일 수 있으면 다시 쓸 것.

**외과적 변경:**
- 손대야 할 것만 손댈 것.
- 인접 코드의 "개선"·포매팅·주석 변경 금지.
- 깨지지 않은 것 리팩토링 금지.
- 기존 스타일을 따를 것 — 본인 취향이 달라도.
- 변경된 모든 줄이 사용자 요청에서 직접 도출되어야 함.

**검증 가능한 목표:**
- "validation 추가" → "잘못된 입력에 대한 테스트 작성, 통과시키기"
- "버그 수정" → "버그 재현 테스트 작성, 통과시키기"
- 멀티스텝 작업은 짧은 계획부터: 단계 → 검증 → 단계 → 검증

## Project-specific rules

### Markdown is the source of truth

**모든 일정 데이터의 원본은 마크다운이다.** 시각화·채팅·드래그 편집 모두 결국 마크다운을 수정한다. JSON은 파생물일 뿐.

- 어떤 편집 경로든 최종적으로 `lib/markdown/serialize.ts`를 거쳐 마크다운으로 환원되어야 함
- DB에 저장하는 것은 `markdown` 컬럼(string). `meta`는 검색용 캐시일 뿐 권위 없음
- 사용자가 손으로 쓴 자유 텍스트, 빈 줄, 이모지, 코멘트 보존 필수
- 마크다운 → JSON → 마크다운 라운드트립이 무손실이어야 함 (테스트 필수)

### LLM editing rules

- AI는 마크다운만 수정. JSON 직접 조작 금지.
- 모든 AI 변경은 `tool_use`로 처리. 평문 응답을 파싱해서 일정에 반영하는 코드 금지.
- LLM 응답은 항상 Zod로 검증 후 적용.
- AI 변경은 사용자에게 diff로 보여주고 승인받기 전엔 저장하지 않음.
- 시스템 프롬프트는 `prompts/system.md`에서 관리. 코드 인라인 금지.
- 큰 변경(마크다운 50% 이상 교체)은 자동 적용 금지, 추가 확인 받기.

### Mobile-first

- 디자인·구현 모두 모바일(375px) 기준.
- 터치 타겟 최소 44px.
- 모바일에서 채팅·에디터·시각화는 탭 전환. 데스크탑은 동시에 보임.
- 새 기능 추가 시 모바일에서 먼저 확인.

### Theming

- 모든 시각화 컴포넌트는 CSS 변수로 색·간격·타이포 노출.
- 하드코딩된 색·폰트 금지 (Tailwind 사용 시에도 의미적 토큰만 — `bg-primary` OK, `bg-blue-500` 금지).
- 테마 CSS는 `themes/{view}/{name}.css`에 분리.

### Supabase rules

- 클라이언트에서 service role key 사용 절대 금지. anon key + RLS만.
- 모든 테이블에 RLS 정책 필수. 정책 없는 테이블 생성 금지.
- DB 쿼리는 `lib/supabase/`를 통해서만. 컴포넌트에서 직접 호출 금지.

### State management

- 일정 상태는 Zustand 스토어 하나로 일원화.
- 에디터·시각화·채팅이 모두 같은 스토어를 구독.
- 스토어 → DB 동기화는 debounce 1초.

## Banned patterns

- `pages/` 디렉토리 (App Router 전용)
- 클라이언트에서 Anthropic API 직접 호출 (반드시 `/api/chat` 경유)
- 클라이언트에서 Supabase service role key 사용
- LLM 평문 응답을 정규식으로 파싱해 일정 수정
- 일정 데이터를 JSON으로만 저장하고 마크다운을 재생성하는 패턴
- `any` 타입 (불가피하면 `// FIXME:` 주석으로 사유)
- 인라인 스타일·하드코딩 색상 (`style={{color: '#fff'}}`, `bg-blue-500` 등)
- 200줄 넘는 컴포넌트 (분리하기)

## Definition of done

변경 후 다음을 모두 통과해야 "완료":

1. `pnpm typecheck` 통과
2. `pnpm lint` 통과
3. 모바일 뷰포트(375px)에서 동작 확인
4. 마크다운 라운드트립 깨지지 않음 (관련 기능일 경우)
5. 어떤 시나리오를 수동 확인했는지 보고
6. 새 라이브러리 추가 시 사유 보고
